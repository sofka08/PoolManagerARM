# Руководство пользователя #
**PoolManager** – настольное приложение для учёта клиентов, управления абонементами и расписанием занятий бассейна. Приложение помогает вести список клиентов и тренеров, выдавать абонементы, записывать и отслеживать занятия, а также получать простую статистику.

## Зачем нужен проект ##  
PoolManager создан для упрощения рутинной работы персонала бассейна:
- хранить данные клиентов и контактную информацию в одном месте;
- удобно управлять абонементами и оставшимися посещениями;
- быстро записывать клиентов на занятия и отслеживать статус (запланировано / проведено / отменено);
- получать базовую статистику по тренерам и абонементам.
  
## Возможности ##
- регистрация и редактирование клиентов;
- регистрация и редактирование тренеров;
- создание шаблонов абонементов и выдача клиентам;
- запись на занятия (расписание) с проверкой конфликтов по тренеру и зоне;
- списание/возврат посещений по абонементам;
- простая статистика: количество клиентов, проведённых занятий, доход;
- экспорт/резервное копирование базы (при желании).

## Технологии ##
- Язык: C#
- Платформа: .NET / .NET Framework (указать точную версию в проекте)
- Тип приложения: настольное (Windows Forms)
- Среда разработки: Visual Studio (рекомендуется 2019/2022)
- Хранение данных: локальная SQLite база (my_database.db)

## Требования ##  
- ОС: Windows 10/11
- Установленная версия .NET, соответствующая проекту (проверь .csproj)
- Visual Studio для запуска из исходников (если запускаешь сборку — достаточно exe)

## Установка и запуск ##  
1. Склонируйте репозиторий или скачайте ZIP и распакуйте.
2. Откройте файл решения PoolManager.sln в Visual Studio.
3. Дождитесь восстановления зависимостей (NuGet), если требуется.
4. Перед первым запуском:
   - Если рядом с приложением нет файла my_database.db, приложение попытается создать базу и заполнить начальными данными автоматически.
   - Если вы хотите использовать готовую базу шаблона, положите файл `my_database_template.db` в папку запуска и переименуйте в `my_database.db`.
5. Запустите проект (F5) или запустите exe из папки bin/Debug (при сборке).

## Частые проблемы и решения ##
- Проблема: «Приложение не запускается — не найден my_database.db»
  - Решение: либо запустить приложение — оно создаст базу автоматически, либо скопировать `my_database_template.db` в папку запуска и переименовать в `my_database.db`.
- Проблема: «Конфликт времени/накладки занятия»
  - Решение: при добавлении записи приложение проверяет перекрытие по тренеру и зоне; выберите другое время или тренера.
- Проблема: «Абонемент не списывается»
  - Решение: проверьте, что при записи указан корректный clientSubscriptionId и что срок/оставшиеся посещения > 0.
- Проблема: «Не сохраняются изменения»
  - Решение: проверьте права на запись в папку с my_database.db и наличие свободного места на диске.
