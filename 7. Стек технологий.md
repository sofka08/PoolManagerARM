# Обоснование выбора стека технологий

Для разработки АРМ администратора бассейна был выбран современный и надежный стек технологий, обеспечивающий высокую производительность и стабильность работы в локальной среде.

## 1. Язык программирования

**C#** выбран в качестве основного языка разработки по следующим причинам:
*   **Стандарт индустрии:** Является основным языком для разработки корпоративного и прикладного ПО под управлением ОС Windows.
*   **Строгая типизация:** Снижает количество логических ошибок на этапе написания кода и облегчает работу с объектами (Клиенты, Тренеры, Занятия).
*   **Производительность:** Прямое взаимодействие с .NET Framework обеспечивает быстрый отклик интерфейса, что критично для работы администратора на рецепции.

## 2. Фреймворк и графический интерфейс

В качестве платформы разработки используется **Windows Forms (.NET Framework 4.8)**. 

**Обоснование выбора:**
*   **Скорость разработки UI:** Наличие визуального конструктора позволяет быстро создавать сложные формы с использованием стандартных компонентов Windows.
*   **Стабильность:** Технология WinForms проверена временем и гарантирует корректное отображение элементов интерфейса на большинстве рабочих станций.
*   **Низкие системные требования:** Приложение потребляет минимальное количество оперативной памяти, что позволяет использовать его даже на устаревших офисных ПК.
*   **Интеграция:** Полная совместимость с ОС Windows 10 и Windows 11 без установки дополнительных компонентов.

## 3. Система управления базами данных (СУБД)

В данной системе используется **SQLite**.

**Преимущества SQLite для данного проекта:**
*   **Автономность (Serverless):** База данных не требует установки сервера. Весь массив данных хранится в одном файле `my_database.db`, который легко переносить или резервировать.
*   **Реляционная структура:** Позволяет строить сложные связи между таблицами (например, связь «Один-ко-многим» между клиентом и его абонементами).
*   **Использование SQL:** Позволяет выполнять сложные аналитические запросы для модуля статистики, которые невозможно реализовать в простых файлах.
*   **Целостность данных:** Поддержка механизмов `FOREIGN KEY` гарантирует, что записи не будут повреждены при удалении связанных сущностей.

## 4. Инструменты тестирования

Для проверки корректности работы программного обеспечения используются:

* тест-кейсы, основанные на сценариях использования системы;
* ручное функциональное тестирование основных возможностей приложения.
Данный подход является достаточным для небольшого локального приложения и позволяет проверить корректность работы всех ключевых функций.

## 5. Вывод

Выбранный технологический стек (**C#, WinForms, SQLite**) является оптимальным для создания локального автоматизированного рабочего места.  
Он обеспечивает баланс между скоростью разработки, безопасностью персональных данных клиентов и удобством дальнейшей поддержки программного продукта.
